 PUNTOS CLAVE DE LA SOLUCIÓN
═══════════════════════════════════════════════════════════════════════════════

🔑 HashRouter vs BrowserRouter
   • BrowserRouter: Requiere server-side routing (NO funciona en GitHub Pages)
   • HashRouter: URLs con # que funcionan sin servidor (✅ GITHUB PAGES)
   • Ejemplo: https://site.com/#/cart → funciona en GitHub Pages

🔑 Base Path en Vite
   • GitHub Pages sirve desde: /usuario/repositorio/
   • Todos los assets deben referenciarse desde ese base
   • Vite configura automáticamente con base: '/Fullstack-Challenge---E-commerce/'

🔑 404.html para SPA
   • GitHub Pages es static-only
   • No puede hacer routing en servidor
   • Usa 404.html para redirigir todas las rutas a index.html
   • HashRouter maneja el routing en el cliente

🔑 Code Splitting y Minificación
   • Cada página se carga bajo demanda (lazy loading)
   • React vendor separado para mejor caching
   • Terser minifica agresivamente
   • Resultado: App rápida y optimizada
